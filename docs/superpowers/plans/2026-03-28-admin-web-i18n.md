# NautilusTrader Admin Web Internationalization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add frontend-only internationalization to `apps/admin-web` with initial support for `en` and `zh-CN`, including browser-language detection, manual locale switching, persisted preference, and localized UI-owned copy without changing backend contracts or route URLs.

**Architecture:** Add a small typed i18n layer under `apps/admin-web/src/shared/i18n/`, using an English base catalog and a matching `zh-CN` catalog. Wire the provider at the app root, localize shell and shared components first, then migrate page-owned copy route by route while keeping raw backend values and server error payloads untranslated. Replace persisted route labels in workspace storage with route identity so recent views always render in the active locale.

**Tech Stack:** React 19, TanStack Router, TanStack React Query, Vite, Vitest, TypeScript, browser `localStorage`
